# Examples Specification

## Intent

Store runnable and readable examples that demonstrate common agent workflows.

## Role

- Provide sample scan requests.
- Provide example analytics payloads.
- Provide example release-note artifacts.
- Provide small fixture references for demos.

## Inputs

- Public repository URLs.
- Sample REST requests.
- Sample MCP tool calls.
- Sample generated artifact bundles.

## Outputs

- Example request JSON.
- Example response JSON.
- Example reports and diagrams when implementation begins.

## Design Rules

- Examples must avoid private repositories and secrets.
- Examples should be small enough for quick local validation.
- Generated outputs should clearly identify that they are examples.
- Keep examples aligned with current API and MCP schemas.
